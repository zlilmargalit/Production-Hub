The root cause, fixed once

Almost every screenshot shows the same failure: content wider than the viewport, so the page scrolls sideways and clips its edge (worse in RTL, where the clip eats the right side). Four global rules stop it everywhere — then each screen gets its specific layout fix.

overflow-x: clip
On html, body + every page container. Nothing can push the page sideways.
min-width: 0
On all flex/grid children, so long Hebrew strings wrap instead of forcing overflow.
overflow-x: auto
Tab bars & the account cluster become scroll tracks instead of overflowing.
min-height: 44px
Every button, checkbox & toggle meets the iOS thumb-target floor.
01

Show detail — the expanded card

Source: IMG_4689–4690. Sound/Lighting blocks, rental fields and the Taxi/Van/Self toggle were packed into widths they couldn't hold, clipping on the right.

Tap Technical / Logistics · toggle Sound & Lighting
◆ What was breaking
  • Sound / Lighting coordination + their rental fields fought for one row and clipped.
  • Taxi / Van / Self pills were tiny and uneven — below the 44px target.
  • Inputs (Driver, Time, Rental company) ran past the right edge.
● The fix
  • Each modality is a self-contained block — switch on top, its rental fields stacked underneath. Off → fields hide.
  • Transport is a 3-up equal grid that collapses to a single column under 360px.
  • Field pairs are grid 1fr 1fr, stacking when there isn't room.
Rules to apply
/* technical coordination — one block per modality */
.tc-fields { display:grid; gap:12px; }
.tc-mod:not(.on) .tc-fields { display:none; }

/* transport: equal thirds, stack on tiny screens */
.seg3 { display:grid;
  grid-template-columns:1fr 1fr 1fr; gap:8px; }
@media (max-width:359px) {
  .seg3 { grid-template-columns:1fr; } }

/* field pairs reflow */
.pair { display:grid;
  grid-template-columns:1fr 1fr; gap:10px; }
.seg3 button { min-height:44px; }

Shows list — card anatomy

Source: IMG_4691. Edit / Delete sat on top of the title, the meta line ran off-edge, and the Brief/PDF + Invoice/Receipt rows were cramped.

Tap the ⋯ on a card to open the action sheet
◆ What was breaking
  • + Edit Delete overlapped the Hebrew title and were untappable.
  • Date · venue · time · crew ran as one un-wrapping line.
  • Footer doc buttons and Invoice/Receipt checks collided.
● The fix
  • Card actions collapse to one ⋯ button → bottom sheet (Edit, Add task, Duplicate, Delete).
  • Meta line is a flex-wrap row that flows onto 2 lines cleanly.
  • Footer wraps: doc buttons left, status chips right, full 44px targets.
Rules to apply
/* one menu button replaces the action row */
.scard-toprow { display:flex;
  justify-content:space-between; }
.scard-more { width:44px; height:38px; }

/* meta + footer wrap instead of overflow */
.scard-meta, .scard-foot {
  display:flex; flex-wrap:wrap;
  gap:12px; min-width:0; }
.scard h3 { text-wrap:balance; }
03

Crew & Event Types — squashed actions

Source: IMG_4692–4693. The CREW / FIELDS / CHECKLIST / X row crushed the Hebrew type name; crew cards clipped on the right and tags overflowed. Use the “Action UI” switch in the phone to compare three patterns.

Switch Sheet / Stacked / Scroll · flip to Members
◆ What was breaking
  • Four actions + RTL name in one row → name squashed, X clipped off-screen.
  • Crew member cards bled past the viewport; Edit/Delete were hover-only (no hover on touch).
  • Event-type tags overflowed in a single non-wrapping line.
● Three patterns to choose from
  • Sheet (recommended) — name gets full width; actions live behind ⋯ in a bottom sheet.
  • Stacked — a 2-col button grid below the name, Delete full-width.
  • Scroll — a horizontal chip strip when you want them all visible.
  • Crew tags flex-wrap; Edit/Delete become a persistent footer.
Rules to apply
/* A — sheet (default): hide inline actions */
.mode-sheet .etactions { display:none; }
.mode-sheet .etcard-more { display:grid; }

/* B — stacked 2-col grid */
.mode-stack .etactions { display:grid;
  grid-template-columns:1fr 1fr; gap:8px; }

/* C — horizontal scroll strip */
.mode-scroll .etactions { display:flex;
  overflow-x:auto; gap:8px; }

/* tags always wrap */
.crew-tags { display:flex; flex-wrap:wrap; }
04

Tasks — RTL Hebrew rows

Source: IMG_4694. Date pills and event-type tags collided, and mixing RTL text with LTR layout produced ragged, mismatched baselines.

Tap a checkbox to complete · ⋯ for task actions
◆ What was breaking
  • Title, date pill and tag shared one cramped baseline and overlapped.
  • LTR row layout + RTL text → tags floated to the wrong side.
  • Edit/Delete (hover) overlapped the task text.
● The fix
  • Row = leading 44px checkbox · content block · trailing ⋯ — fixed LTR scaffold.
  • Inside, the text block is dir="rtl" and right-aligned; meta chips flex-wrap and align right.
  • Per-element direction keeps the scaffold steady while Hebrew reads correctly.
Rules to apply
/* stable LTR scaffold, RTL content inside */
.trow { display:flex; align-items:flex-start;
  gap:12px; }
.ttitle { text-align:right; }
.ttitle[dir="rtl"] /* in markup */

/* chips wrap and hug the right edge */
.tmeta { display:flex; flex-wrap:wrap;
  justify-content:flex-end; gap:7px; }
.tcheck { width:26px; min-height:26px; }
.tmore { width:44px; }
05

Automations & Recipes

Source: IMG_4695–4696. Connected-app pills and recipe cards were cramped, and the custom-rule builder was visually overwhelming.

Single-column scroll · the builder reads top-down
◆ What was breaking
  • Gmail / Calendar / Drive pills sat on one line and clipped.
  • Recipe cards carried a heavy offset shadow that crowded the text.
  • TRIGGER → CONDITION → ACTION had no clear vertical hierarchy.
● The fix
  • App pills flex-wrap; “Manage integrations” becomes a full-width button.
  • Recipe cards are flat, full-width, with a single accent strip on top.
  • Builder is a vertical pipeline: tagged steps joined by ↓ connectors.
Rules to apply
/* pills wrap, never overflow */
.apps-row { display:flex; flex-wrap:wrap;
  gap:8px; }

/* builder reads top-to-bottom */
.builder { display:flex;
  flex-direction:column; }
.bstep { padding:16px;
  border-top:1px solid var(--border); }
.btag { padding:5px 10px;
  border-radius:6px; /* TRIGGER/COND/ACTION */ }
.recipe .btn { width:100%; }

Teams & Permissions — the matrix

Source: IMG_4697–4698. The Content Permissions table was cut off on the right (View/Edit columns unreachable) and the bottom tabs overlapped the add-task form.

Flip View / Edit toggles · switch the bottom tabs
◆ What was breaking
  • 3-column Area / View / Edit table was wider than the screen — the toggle columns were unreachable.
  • Assigned Tasks / Upcoming Shows / Recent Activity tabs overlapped the form below.
  • The add-task select + input + button were stacked on top of each other.
● The fix
  • Matrix → per-area cards. Each area shows its sub-fields and two big View / Edit toggles (Edit auto-enables View).
  • Bottom tabs become an equal-width segmented control; the form sits clearly beneath it.
  • Add-task row stacks: select on its own line, input + Add on the next.
Rules to apply
/* kill the table — one card per area */
.area { border:1px solid var(--border);
  border-radius:12px; padding:13px 14px; }
.toggles { display:grid;
  grid-template-columns:1fr 1fr; gap:8px; }
.tg { min-height:44px; }

/* tabs: equal segments, no overlap */
.mtabs button { flex:1; min-width:0; }
/* add-task stacks */
.addtask { display:grid; gap:8px; }
